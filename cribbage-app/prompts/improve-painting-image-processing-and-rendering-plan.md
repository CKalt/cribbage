## Problem Statement

### Current issues

1. **No standardized image preparation process.** Cropping has been done ad-hoc with hardcoded percentages, leading to inconsistent results вҖ” some paintings were over-cropped (ornate frame removed), others under-cropped (white margin visible).

2. **CSS adds an extra colored border around paintings.** All rendering locations apply `border-2 ${cardBack.border}` (e.g., `border-amber-200`) around painting cards. Since the painting image already contains its own cream border and ornate frame, the CSS border is redundant and visually wrong вҖ” it adds a colored line that doesn't belong.

3. **No originals archive.** Original uncropped images were not systematically preserved, requiring recovery from git history.

### Desired result

- A **command-line tool** that auto-detects and crops the white background, preserving the cream card body with its rounded corners, ornate frame, and painting.
- **Rendering with no CSS border** for painting cards вҖ” the image IS the card. Only `overflow-hidden` and `rounded` (CSS border-radius) are needed to clip the corners.
- The card should look identical in game play, Admin Panel, deck cut overlay, flying animation, and full-screen preview.

### Reference image

The original uncropped image is at: `originals/skyscraper-paiting-okeeffe.png` (1024Г—1536)

Analysis of white border boundaries:
- **Left**: 126px of white (12.3%)
- **Right**: ~124px of white (12.1%)
- **Top**: 123px of white (8.0%)
- **Bottom**: ~225px of white (14.6%)

The crop algorithm should auto-detect the whiteвҶ’cream transition rather than use fixed percentages, since different images may have slightly different border widths.

### Step 1.2: Create the crop-painting script рҹӨ–

Create `src/crop_painting.py` вҖ” an auto-crop script that:

1. Loads the source PNG from `originals/`
2. Detects the white background boundary by scanning inward from each edge, finding where pixel color transitions from white (RGB all > 250) to cream/card content (any channel < 250)
3. Crops to those boundaries, removing only the white exterior
4. Saves the result to `public/card-backs/` (same filename)

**Algorithm for white detection:**
- Scan inward from each of the 4 edges along multiple sample lines (not just one pixel row/column, to handle rounded corners)
- For each sample line, find the first pixel where `min(R, G, B) < 250`
- Take the **maximum** inset across all sample lines for each edge (conservative crop вҖ” ensures all white is removed even at the widest point)
- Add a small safety margin (2-3px inward) to ensure no white fringe remains

**Usage:**
```bash
source ./source-venv.sh && python src/crop_painting.py originals/skyscraper-paiting-okeeffe.png
```

The script should:
- Print the detected boundaries and crop dimensions
- Save the cropped image to `public/card-backs/<filename>`
- Support processing a single file or all files in `originals/` via a `--all` flag

## Phase 3: Update Painting Card Rendering (No Extra Border) рҹӨ–

The key principle: **for painting cards, the image IS the card.** The CSS should only provide:
- `overflow-hidden` вҖ” clips the image to the container bounds
- `rounded` вҖ” applies border-radius to match the cream card's rounded corners
- Fixed dimensions вҖ” `w-10 h-14` etc. for the card size

The CSS should NOT add:
- `border-2` or any border width
- `${cardBack.border}` (colored border class)
- `${cardBack.bg}` (background color вҖ” the image fills completely)

### Step 3.1: Update CardBack.jsx вҖ” remove border for paintings рҹӨ–

**Current** (line 175):
```jsx
<div className={`${design.bg} ${s.border} ${design.border} rounded ${s.outer} relative overflow-hidden cursor-pointer ${className}`}>
```

**Change to** (for `sceneImage` branch only):
```jsx
<div className={`rounded ${s.outer} relative overflow-hidden cursor-pointer ${className}`}>
```

Remove `${design.bg}`, `${s.border}`, and `${design.border}` from the painting rendering path. The `bg` is unnecessary since the image covers the entire area. The border would add a colored line around the card that doesn't belong.

### Step 3.2: Update CribbageGame.jsx вҖ” deck piles and crib piles рҹӨ–

There are **8 locations** where painting cards are rendered inline in CribbageGame.jsx (2 deck piles Г— 1 + 2 crib piles Г— 4 stack levels). Each currently applies `${cardBack.bg} border-2 ${cardBack.border}`.

For `sceneImage` renders, change the container class to omit bg and border:

**Current pattern:**
```jsx
className={`absolute ${cardBack.bg} border-2 ${cardBack.border} rounded overflow-hidden ...`}
```

**New pattern** (conditionally for paintings):
```jsx
className={`absolute ${cardBack.sceneImage ? '' : `${cardBack.bg} border-2 ${cardBack.border}`} rounded overflow-hidden ...`}
```

This restores the conditional that was previously removed, but now the painting image includes its own cream border so no CSS border is needed.
